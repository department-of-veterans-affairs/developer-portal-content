We updated 2 endpoints in version 1 of the Benefits Claims API to fix an issue where the API returned null or incorrect metadata values for uploaded supporting documents. 

**Updated endpoints include:**
- GET /claims
- POST /forms/526/{id}/attachments

**These endpoints now return correct metadata** for supporting documents:
- `filename`
- `header_hash`
- `md5`
- `uploaded_at`

This update ensures you can reliably match supporting document IDs with their corresponding uploaded files.
