Version 1 of the Clinical Health API is now available in the **production environment**.

**What's new?**

Authorization provider:
* The API now uses Microsoft Entra ID for authorization, replacing the previous single sign-on internal (SSOi) system
* JSON web tokens (JWTs) are issued without an introspection endpoint

Token behavior and policy:
* Access tokens now expire between 60-90 minutes for enhanced security
* Access token lifetime is variable by design
* Refresh tokens expire 10 hours after issuance, regardless of usage

Version 0 of the Clinical Health API will be **deactivated in April 2026.**
